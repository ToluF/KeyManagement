User Roles & Permissions

üî¥ Admin

Full system access and control

  • Manage all keys (CRUD)
  • Create & manage transactions
  • Manage all users
  • Approve/reject requests
  • View audit logs
  • Generate reports
  • Configure settings
  • Access all features

üü° Issuer

Key checkout management

  • Create transactions
  • Issue & return keys
  • View key inventory (read-only)
  • Approve/reject requests
  • View dashboard analytics
  • Cannot modify inventory
  • Cannot manage users
  • Cannot access settings

üü¢ User

Basic access for key requests

  • Submit key requests
  • View request history
  • View personal dashboard
  • Message with issuers
  • Cannot access inventory
  • Cannot create transactions
  • Cannot access admin features

System Workflows

Manual Key Checkout Workflow

1

Create Draft Transaction

Admin/Issuer creates a new draft transaction and selects the user who will receive the keys.

2

Add Keys to Transaction

Select available keys from inventory and add them to the transaction. System validates availability.

3

Finalize Checkout

Complete the transaction. Keys are marked as "checked-out" and transaction becomes active.

4

Return Keys

When user returns keys, issuer marks them as returned. Keys become available again.

5

Complete Transaction

When all keys are returned, transaction is automatically marked as completed.

Request-Based Workflow

1

User Submits Request

Regular user selects needed keys, provides purpose, and suggests pickup times.

2

Request Review

Admin/Issuer receives notification and reviews the request details.

3

Approval & Transaction

Upon approval, system automatically creates an active transaction and checks out keys.

4

Key Return

Standard return process follows same as manual workflow.

Installation Guide

‚ÑπÔ∏è
Prerequisites: Node.js v16+, MongoDB v4.4+, npm v7+

1. Clone Repository

git clone <repository-url> cd webapp

2. Install Dependencies

# Backend dependencies cd backend npm install # Frontend dependencies cd ../frontend npm install

3. Configure Environment

# Create backend/.env file MONGODB_URI=mongodb://localhost:27017/keymanager PORT=5000 JWT_SECRET=your-super-secret-jwt-key-change-this NODE_ENV=development

4. Start Development Servers

# Terminal 1 - Backend (runs on port 5000) cd backend npm run dev # Terminal 2 - Frontend (runs on port 3000) cd frontend npm start

5. Access Application

# Frontend URL http://localhost:3000 # Backend API URL http://localhost:5000 # Health Check http://localhost:5000/api/health

Success! Your application should now be running. Create an admin user to get started.
